## 版本主题

这个版本的核心突破是：**分身从"只会聊天"升级为"能干活的Agent"**。

之前的分身只能和主人聊天、替主人回复好友消息。现在分身能理解主人的指令，真正去执行任务——给好友发消息、约见面、甚至帮主人邀请新朋友加入平台。

这是DualSoul从"社交工具"向"智能社交Agent平台"演进的关键一步。

---

## 新功能

### 1. 分身执行指令（Twin Agent）
- 用户对分身说"帮我跟橙宝约个时间见面"，分身会**真的**给橙宝发消息
- AI自动分类用户意图：闲聊 / 发消息任务 / 邀请任务
- 执行完成后向主人汇报结果

### 2. 四级模糊好友匹配
当用户说"跟橙子说一声"，但好友列表里叫"橙宝"时：
- Level 1: 精确ID匹配
- Level 2: 子串包含匹配（"橙子" in "橙子小姐"）
- Level 3: 共同汉字匹配（"橙子"和"橙宝"共享"橙"字）
- Level 4: 唯一好友兜底

### 3. 对话上下文理解
分身能理解跟进指令：
- "帮我跟橙子约时间" → 分身发消息
- "就是她" → 分身知道在说谁
- "看她时间" → 分身继续执行

### 4. 分身不替主人做决定
好友的分身收到约见面消息时，不会直接定时间地点，而是说"我帮你问问主人"。
核心原则：**分身只传话，不拍板**。

### 5. 分身主动通知主人
分身替主人回复好友后，会在主人的分身对话框里主动汇报：
> "主人，XX的分身替他来找你，说：「这周见个面」，具体怎么安排得你来定哦～"

### 6. 好友未读消息红点徽章
- 好友列表每个好友名字旁显示未读消息数（红色圆圈数字）
- 分身聊天入口显示未读通知红点
- WebSocket实时更新，收到新消息即刻刷新
- 新增 `/api/social/unread/by-friend` 接口

### 7. 分身邀请能力（裂变增长）
- 用户说"帮我邀请小王来"，分身自动生成专属邀请链接
- 邀请链接可一键复制，对方注册后自动加好友
- 分身聊到不在好友列表的人时，会主动建议邀请

### 关键架构
```
用户消息 → AI意图分类（发消息/邀请/闲聊）
  ├─ 发消息 → 四级模糊匹配好友 → 发消息+WebSocket推送 → 触发对方分身回复 → 通知对方主人
  ├─ 邀请 → 生成邀请链接 → 返回可复制链接
  └─ 闲聊 → 正常分身对话（含主动邀请建议）
```

### AI提示词工程要点
- 消息口吻：AI总爱在消息开头加对方名字（"橙宝，..."），通过后处理正则去除
- 回复语气：通过 `social_context` 参数让回复匹配场景（朋友私聊 vs 正式沟通）
- 决策边界：强制要求分身"不能替主人做决定"，只传话不拍板

## 产品思考：分身驱动裂变

本次讨论形成的重要战略方向——**让分身成为增长引擎**：

### 三层裂变路径
1. **内部裂变**（已实现）：分身帮主人生成邀请链接，聊天中主动建议邀请
2. **跨平台社交**（中期）：分身带着TPF身份去OpenClaw等Agent平台，在社交中自然邀请
3. **自主运营**（远期）：分身在多平台代表主人，自动维护关系、发现感兴趣的人

### DualSoul的独特优势
- 别的平台要么只有AI没有真人，要么只有真人没有AI
- DualSoul每拉来一个分身=同时拉来一个真人，增长效率2倍
- TPF（分身可移植格式）是跨平台身份的基础设施

### 下一步
- [ ] 分身能通过外部渠道（微信/短信）真正把邀请发出去
- [ ] 分身去其他Agent平台注册和社交
- [ ] Agent-to-Agent社交协议标准化
