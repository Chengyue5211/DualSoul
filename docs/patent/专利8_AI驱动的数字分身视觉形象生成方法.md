## 一、技术领域

本发明属于计算机视觉、AI图像生成和数字人领域，具体涉及在双身份社交网络中，基于用户真人照片通过AI风格迁移自动生成与用户视觉相关但风格明显区分的数字分身头像的方法和系统。

---

## 二、背景技术

现有数字分身/虚拟形象系统存在以下问题：

（一）**通用头像生成**（Bitmoji、Zepeto、Ready Player Me）：用户手动选择五官特征拼装虚拟形象，耗时长、与真人相似度低、缺乏个性化风格。

（二）**AI换脸/美颜**（FaceApp、Meitu、Snapchat滤镜）：基于真人照片进行变形或美化，但结果仍然是"真人照片的变体"，无法产生明确的"数字分身"视觉区分。

（三）**AI艺术风格迁移**（Prisma、AI绘画工具）：将照片转换为艺术风格，但与社交网络中的双身份概念无关联，无法在社交图谱中作为独立身份的视觉标识。

（四）**手动上传**：用户自行制作或上传分身头像，门槛高，大量用户选择跳过，导致分身形象缺失。

**空白地带**：没有一个系统能在用户注册时自动从真人照片生成风格化的数字分身头像，并将其嵌入双身份社交网络的身份体系中，使真人头像和分身头像在视觉上既相关又明显区分。

---

## 三、发明内容

### 3.1 技术问题

本发明解决的技术问题是：如何在双身份社交网络中，自动从用户的真人照片生成风格化的数字分身头像，降低分身形象创建门槛，同时确保分身头像与真人头像在视觉上"像同一个人的不同存在形态"。

### 3.2 技术方案

#### 3.2.1 双层头像架构

在双身份社交网络中，每个用户拥有两个独立的视觉标识：
- **真人头像（avatar）**：用户真实照片，用于 `sender_mode='real'` 身份
- **分身头像（twin_avatar）**：AI风格化后的照片，用于 `sender_mode='twin'` 身份

两个头像在数据库中独立存储，在社交界面中根据当前身份模式自动切换显示。

#### 3.2.2 注册流程中的头像生成

在用户注册引导流程中集成头像生成步骤：

1. **拍照/上传**：用户通过前置摄像头拍摄自拍照（480×480，居中裁剪），或从相册上传照片
2. **真人头像确认**：拍摄的照片直接作为真人头像
3. **分身风格选择**：用户从预设风格列表中选择一种AI风格
4. **AI生成**：系统将真人照片发送给AI图像风格迁移引擎，生成风格化的分身头像
5. **预览确认**：用户预览生成结果，可重新选择风格或跳过使用CSS滤镜方案

#### 3.2.3 多风格预设体系

系统提供多种预设风格，每种风格通过风格迁移模型的不同参数实现：

| 风格标识 | 名称 | 视觉效果 |
|----------|------|----------|
| `anime` | 动漫 | 日式动漫风格的人物形象 |
| `3d` | 3D立体 | 3D渲染风格的立体头像 |
| `cyber` | 未来科技 | 赛博朋克风格，带科技感光效 |
| `clay` | 黏土 | 黏土/橡皮泥质感的可爱形象 |
| `pixel` | 像素 | 像素艺术风格 |
| `ink` | 水墨 | 中国水墨画风格 |
| `retro` | 复古漫画 | 美式复古漫画风格 |

#### 3.2.4 异步生成与轮询架构

由于AI图像生成耗时较长（约15秒），采用异步任务架构：

1. **提交任务**：客户端向服务端发送生成请求（含base64图片和风格标识）
2. **服务端提交AI任务**：调用图像生成API，获取异步任务ID
3. **轮询等待**：服务端周期性轮询任务状态（每2秒一次，最长60秒）
4. **获取结果**：任务完成后，下载生成的图片，保存为分身头像
5. **返回客户端**：将新头像URL返回给前端，实时更新显示

```
POST /api/identity/avatar/generate
{
  "image": "base64编码的真人照片",
  "style": "anime"  // 风格标识
}
→ Response: { "success": true, "url": "/static/avatars/xxx.png", "style": "anime" }
```

#### 3.2.5 降级方案：CSS滤镜

当AI图像生成服务不可用时，系统自动降级为CSS滤镜方案：
- 使用同一张真人照片
- 通过CSS滤镜（`hue-rotate`、`saturate`、`contrast`等）创建视觉差异
- 7种预设滤镜：赛博朋克、霓虹、素描、波普、梦幻、冰蓝、原版

此降级方案确保所有用户都能拥有视觉上有区分的分身头像，无论AI服务是否可用。

#### 3.2.6 随时可替换

用户在任何时候都可以重新生成或替换头像：
- **更换真人头像**：从相册上传新照片
- **更换分身头像**：手动上传或重新AI生成
- **切换风格**：选择不同的AI风格重新生成

Profile页面提供三个操作入口，降低操作门槛。

### 3.3 有益效果

（一）将分身头像创建门槛从"手动制作"降低为"拍照一键生成"，大幅提高分身形象完成率。

（二）AI风格化确保分身头像与真人头像"像同一个人"但"明显不同"，在社交界面中用户能直觉区分对方当前是真人还是分身在说话。

（三）多风格预设满足不同用户的审美偏好。

（四）CSS滤镜降级方案确保功能在任何条件下可用。

（五）随时可替换机制尊重用户对自身形象的控制权。

## 四、权利要求书

1. 一种在双身份社交网络中自动生成数字分身头像的方法，其特征在于：基于用户的真人照片，通过AI图像风格迁移技术生成风格化的分身头像，并将真人头像和分身头像分别关联到用户的真人身份节点和分身身份节点，使同一用户的两种身份在视觉上既相关又明显区分。

2. 根据权利要求1所述的方法，其特征在于：在用户注册引导流程中集成头像生成步骤，包括拍照/上传、真人头像确认、分身风格选择、AI生成和预览确认五个步骤。

3. 根据权利要求1所述的方法，其特征在于：提供多种预设风格（动漫、3D、未来科技、黏土、像素、水墨、复古漫画等），每种风格通过图像风格迁移模型的不同参数实现，用户可选择偏好风格并随时更换。

4. 根据权利要求1所述的方法，其特征在于：采用异步任务架构处理AI图像生成，包括提交任务、轮询等待和结果获取三个阶段，并在AI服务不可用时自动降级为CSS滤镜方案。

5. 一种双身份社交网络中的头像管理系统，其特征在于：每个用户独立维护真人头像和分身头像两个视觉标识，支持随时更换（上传、AI重新生成、切换风格），在社交界面中根据消息的`sender_mode`字段自动显示对应身份的头像。

## 五、在先技术对比

| 在先技术 | 与本发明的区别 |
|----------|---------------|
| Bitmoji / Zepeto | 手动拼装虚拟形象，非基于真人照片AI生成 |
| FaceApp / Meitu美图秀秀 | 美颜/换脸，结果仍是"真人照片变体"，无双身份概念 |
| Prisma / AI艺术滤镜 | 通用艺术风格迁移，与社交网络身份体系无关联 |
| Ready Player Me | 3D虚拟形象，需手动创建，非一键AI生成 |
| Seedance / 即梦 | 通用AI图像/视频生成，不嵌入双身份社交图谱 |
| Snapchat AR滤镜 | 实时增强现实，临时效果，不生成永久头像 |

本发明的独创性：首次在双身份社交网络中实现基于真人照片的数字分身头像AI自动生成，将生成结果嵌入双身份社交图谱的身份体系中，并提供多风格预设、异步生成、CSS降级和随时替换的完整方案。
