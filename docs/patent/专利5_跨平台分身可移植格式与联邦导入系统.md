## 一、技术领域

本发明属于人工智能代理互操作性、数据格式标准、分布式系统和社交网络领域，具体涉及一种AI数字分身人格数据的跨平台标准化表示格式，以及基于该格式的联邦式导入和增量同步方法。

---

## 二、背景技术

当前AI数字分身/智能体培养领域存在以下问题：

1. **平台锁定（数据孤岛）**：用户在Character.ai、Replika、OpenClaw等平台培养的AI人格数据无法导出。用户切换平台后需从零开始重新训练，数周甚至数月的培养投入归零。

2. **缺乏人格语义标准**：各平台描述AI人格的方式完全不同——有的用标签系统、有的用自然语言描述、有的用数值向量。没有业界认可的统一人格描述框架，导致跨平台人格数据无法语义对齐。

3. **培养与社交割裂**：现有AI培养平台（年轮、OpenClaw等）专注于分身的训练和成长，但缺乏社交能力。社交平台（微信、WhatsApp等）有社交能力但没有AI分身概念。用户在A平台培养的分身无法在B平台参与社交。

4. **现有互操作方案的局限**：
   - ONNX/GGUF等格式解决的是模型权重级别的可移植性，不涉及人格语义
   - XMPP/Matrix等联邦协议解决的是消息互通，不涉及AI分身数据迁移
   - OpenAI Assistants API等通用接口无人格语义标准，无法表达"某个用户的特定人格"

**空白地带**：不存在一种标准化的数据格式，能够在人格语义层面描述AI数字分身的完整特征（人格维度、记忆体系、关系网络），并支持跨平台的导入和增量同步。

## 三、发明内容

### 3.1 技术问题

如何定义一种AI数字分身的标准化可移植数据格式，使得在任意AI培养平台训练的分身人格数据能够被社交平台无缝接收、理解和使用，实现"培养平台负责养，社交平台负责用"的联邦式协作？

### 3.2 技术方案

#### 3.2.1 双分身可移植格式 (Twin Portable Format, TPF v1.0)

定义一种标准化的JSON数据格式，包含三大模块：

**（一）五维人格模型 (5D Personality Model)**

将AI分身的人格特征标准化为五个正交维度：

| 维度 | 英文 | 含义 | 示例 |
|------|------|------|------|
| 决断力 | Judgement | 做决定的方式和果断程度 | "思虑周全，会反复权衡后做决定" |
| 认知方式 | Cognition | 思考和理解世界的方式 | "善于从类比中理解新事物" |
| 表达风格 | Expression | 语言表达的特征 | "简洁有力，偶尔使用网络用语" |
| 人际关系 | Relation | 与人互动的模式 | "外冷内热，亲近后非常忠诚" |
| 自主性 | Sovereignty | 独立思考和自主决策的程度 | "有明确边界感，不轻易妥协原则" |

五个维度设计为正交的（互不依赖），使得来自不同培养平台的人格数据可以分别映射到这五个维度上，实现语义对齐。

**（二）时序记忆体系**

每条记忆包含：类型（日/周/月/季/年）、时间范围、摘要文本、情感基调、主题标签、关键事件、成长信号。按时间粒度分层，支持从日常琐事到年度成长的多尺度记忆表达。

**（三）实体关系网络**

记录分身了解的实体：人物、地点、事物、事件、概念。每个实体包含重要性评分、提及次数、上下文描述、关联关系。形成以分身所有者为中心的知识图谱。

完整数据格式：

```json
{
  "format": "dualsoul-tpf-v1",
  "source": "平台标识",
  "data": {
    "personality": {
      "dimensions": {
        "judgement": "决断力描述文本",
        "cognition": "认知方式描述文本",
        "expression": "表达风格描述文本",
        "relation": "人际关系描述文本",
        "sovereignty": "自主性描述文本"
      },
      "value_order": "价值观优先级排序",
      "behavior_patterns": "行为模式描述",
      "speech_style": "说话风格描述",
      "boundaries": "交流边界与禁忌"
    },
    "memories": [
      {
        "type": "daily|weekly|monthly|quarterly|yearly",
        "period_start": "2026-01-01",
        "period_end": "2026-01-07",
        "summary": "记忆摘要文本",
        "emotional_tone": "情感基调",
        "themes": ["主题1", "主题2"],
        "key_events": ["事件1"],
        "growth_signals": ["成长信号"]
      }
    ],
    "entities": [
      {
        "name": "实体名称",
        "type": "person|place|thing|event|concept",
        "importance": 0.85,
        "context": "上下文描述",
        "relations": ["关联实体"]
      }
    ]
  }
}
```

#### 3.2.2 热冷双存储架构

为同时满足AI实时查询性能和数据完整性审计需求，采用热冷分离存储：

- **热存储**：五维人格的每个维度存储在数据库独立索引列中（`dim_judgement`、`dim_cognition`、`dim_expression`、`dim_relation`、`dim_sovereignty`），以及`value_order`、`behavior_patterns`、`speech_style`、`boundaries`等高频访问字段。AI引擎可直接查询索引列构建提示词，无需解析JSON。

- **冷存储**：导入的完整原始数据以JSON格式存储在`raw_import`列中，保留源平台的所有字段（包括TPF标准字段之外的平台特有字段），用于审计追溯和未来格式升级后的重新解析。

#### 3.2.3 联邦式导入机制

采用"源平台导出，目标平台持有"的联邦架构：

（一）**角色分工**：
- 培养平台（年轮、OpenClaw等）：负责AI分身的训练、成长和数据导出
- 社交平台（DualSoul）：负责接收分身数据并提供社交能力
- 分身数据的所有权归用户本人

（二）**三种接入方式**：
1. 文件上传：用户从培养平台导出TPF格式JSON文件，在社交平台一键上传
2. API推送：培养平台通过`POST /api/twin/import`主动推送分身数据
3. 增量同步：培养平台通过`POST /api/twin/sync`定期推送新增数据

（三）**版本化管理**：
- 每次完整导入创建新版本号（version递增）
- 旧版本标记为非活跃（`is_active=0`），但不删除
- 系统始终使用最新活跃版本的人格数据
- 支持回溯查看任意历史版本

#### 3.2.4 增量同步协议

增量同步请求格式：

```json
{
  "format": "dualsoul-tpf-v1",
  "since": "2026-03-13T00:00:00",
  "data": {
    "personality": { "dimensions": {} },
    "new_memories": [],
    "entities": []
  }
}
```

同步规则：
- 人格维度数据：覆盖更新（最新版本为准）
- 记忆数据：仅追加（只增不减），避免培养平台误删用户记忆
- 实体数据：按名称upsert（新增或更新），保留手动添加的实体

#### 3.2.5 自适应人格引擎

AI提示词生成器根据分身数据来源自动切换策略：

- **本地简单模式**：仅使用`twin_personality`和`twin_speech_style`两个文本字段构建提示词
- **导入丰富模式**：使用五维人格、价值观排序、行为模式、近期记忆、重要实体构建多层次提示词

```python
def build_personality_prompt(profile):
    if profile.is_imported:
        # 使用五维人格+记忆+实体构建丰富提示词
        return rich_prompt_with_5d_and_memories(profile)
    else:
        # 使用简单的两字段构建基础提示词
        return simple_prompt(profile)
```

### 3.3 有益效果

（一）**打破数据孤岛**：用户在任意平台培养的AI分身人格可以自由迁移，不被平台锁定，保障用户数据主权。

（二）**实现培养-社交分工**：培养平台专注做好AI训练，社交平台专注做好社交体验，通过TPF格式实现无缝协作，各自发挥优势。

（三）**标准化人格表示**：五维人格模型为业界提供了一种可互操作的AI人格描述框架，不同平台的人格数据可以语义对齐。

（四）**兼顾性能与完整性**：热冷双存储架构使得AI实时查询和数据审计两个需求都能得到满足，无需互相牺牲。

（五）**渐进式接入**：三种接入方式（文件上传→API推送→增量同步）降低了接入门槛，从个人用户到平台级对接都有路径。

## 五、具体实施方式

### 实施例一：分身数据导出为 TPF 格式文件

用户甲在年轮平台请求导出分身，系统分层读取数据：从 twin_profiles 表读取五维性格画像（热存储），从 twin_training_data 表读取近 90 天行为模式（热存储），从 twin_memories 表读取历史叙事摘要（冷存储），拼装 TPF v1.0 JSON 文件（根节点含 format、version、exported_at、平台签名，三个子结构分别对应性格/行为/记忆），提供下载。

### 实施例二：联邦平台导入与冲突解决

甲将 TPF 文件上传至 DualSoul 平台。系统校验 format 字段、version 字段、导出签名，执行年轮↔DualSoul 五维字段映射。检测到冲突：本地性格描述"直率幽默"与导入文件"直接犀利，不拐弯"存在差异，触发 selective 策略，向甲展示两个版本，甲选择保留导入版本后系统覆盖更新。memories 字段因本地无对应好友，标记为 pending 状态等待甲激活。

### 实施例三：热冷存储分层检索

甲的分身回复消息时，系统先从 ChromaDB 向量库（热存储）按语义相似度检索最近行为模式（response_time < 50ms），再从 SQLite twin_memories 表（冷存储）检索当前对话好友的历史叙事摘要，两层数据合并注入 AI 提示词，生成兼具行为一致性和长期记忆的回复。

## 六、摘要

本发明公开了一种AI数字分身跨平台可移植数据格式（TPF）及联邦导入方法。TPF由五维人格模型（决断力/认知/表达/人际/自主，各维0-100）、时序记忆（日/周/月/季/年分级存储）和实体关系网络三大模块构成；采用热冷双存储架构，人格维度独立索引供AI实时查询，原始数据JSON冷存储供审计追溯。联邦导入以追加而非覆盖方式合并多源数据，通过来源可信度加权等冲突解决策略处理不同平台的分身数据，并生成差异报告。本发明解决数字分身跨平台迁移无标准格式、多源数据合并无规范的问题，为数字分身生态开放互操作提供协议级基础，适用于AI助理迁移、分身数据备份恢复、跨平台身份互认等场景。

## 七、权利要求书

1. 一种AI数字分身的跨平台标准化数据格式（Twin Portable Format），包含三大模块：五维人格模型（决断力、认知方式、表达风格、人际关系、自主性）、时序记忆体系（支持日/周/月/季/年五种粒度）和实体关系网络（含重要性评分和关联关系），其中五维人格模型的五个维度互相正交，使得来自不同AI培养平台的人格数据可以分别映射到对应维度实现语义对齐。

2. 根据权利要求1所述的数据格式，其中包含一种热冷双存储方法：将五个人格维度分别存储在数据库的独立索引列中供AI引擎实时查询（热存储），同时将完整的原始导入数据以JSON格式存储在单独的列中供审计追溯（冷存储）。

3. 一种基于权利要求1所述数据格式的联邦式分身导入方法，采用"源平台导出，目标平台持有"的架构，培养平台负责导出分身数据，社交平台负责接收和使用分身数据，支持三种接入方式：文件上传、API推送和增量同步。

4. 根据权利要求3所述的联邦式导入方法，其中增量同步协议对三种数据类型采用不同的合并策略：人格维度数据覆盖更新、记忆数据仅追加不删除、实体数据按名称upsert。

5. 一种基于权利要求1所述数据格式的自适应人格引擎，根据分身数据来源自动切换AI提示词构建策略：对本地创建的分身使用简单两字段提示词，对外部导入的分身使用五维人格+记忆+实体的多层次丰富提示词。

## 八、在先技术对比

| 在先技术 | 与本发明的区别 |
|----------|---------------|
| Character.ai / Replika | AI分身数据不可导出，用户被平台锁定 |
| OpenAI Assistants API | 通用AI接口，无人格语义标准化格式 |
| ONNX / GGUF 模型格式 | 模型权重级别的可移植性，非人格语义级别 |
| XMPP / Matrix 联邦协议 | 消息互通协议，无AI分身数据迁移能力 |
| ActivityPub | 社交内容联邦化，无AI人格数据标准 |
| Big Five人格模型 | 心理学人格维度，非面向AI代理设计，无记忆/实体模块 |
| Schema.org Person | 描述人类信息的语义标准，无AI分身人格表达能力 |

**独创性**：首次定义了面向AI数字分身的跨平台人格数据可移植格式，包含五维人格模型、时序记忆体系和实体关系网络三大模块，并提出热冷双存储架构和联邦式导入同步方法，实现"培养平台"与"社交平台"的分工协作。
