## 一、技术领域

---

## 二、背景技术

现有跨语言社交通信方案存在以下问题：

（一）通用机器翻译（Google Translate、DeepL等）：语法准确但丢失个人风格，翻译结果读起来像翻译器产物，不像原始说话人。

（二）实时翻译社交功能（微信翻译、iMessage翻译）：嵌入社交APP的翻译仍为通用翻译，无任何个人风格保真处理。

（三）多语言AI对话（ChatGPT多语言模式）：AI可用多种语言交流，但使用通用风格，不代表特定用户人格。

没有一个系统能在跨语言社交中保持说话人的个人风格——幽默感、语气、正式程度、惯用表达方式。这一空白地带是本发明所要填补的技术问题。

---

## 三、发明内容

### 3.1 技术问题

本发明解决的技术问题是：如何在跨语言社交通信中，利用用户的数字分身人格画像，使翻译结果不仅语言正确，而且保留说话人的个人风格特征。

### 3.2 技术方案

#### 3.2.1 人格保真翻译核心方法

将用户的人格画像（personality profile）注入翻译过程，构建人格保真翻译提示词框架：

```
你是{owner_name}的个人翻译师。
性格特征：{twin_personality}
说话风格：{twin_speech_style}

将以下内容从{source_lang}翻译成{target_lang}。
重要：不要简单翻译词语。改写成{owner_name}如果自然地说{target_lang}
会怎么表达——保留他们的幽默感、语气、正式程度和惯用表达方式。
```

效果对比：

| 类型 | 输入 | 输出 |
|------|------|------|
| 传统翻译 | "哈哈这事儿太逗了！" | "Haha, this is too funny!" |
| 人格保真翻译（幽默直爽风格） | "哈哈这事儿太逗了！" | "LOL this is absolutely hilarious!" |

#### 3.2.2 协议级翻译溯源

在DISP消息格式中增加四个翻译溯源字段：

```json
{
  "content": "翻译后的内容",
  "original_content": "原始内容",
  "original_lang": "zh",
  "target_lang": "en",
  "translation_style": "personality_preserving"
}
```

翻译不变式（5条）：
1. 翻译后的消息必须保留原始内容字段（溯源）
2. 翻译风格标记不可被修改（透明度）
3. 人格画像必须参与翻译过程（人格保真）
4. 目标语言必须是有效的语言代码（语言有效性）
5. 已翻译的消息不可被重复翻译（防重复）

#### 3.2.3 自动跨语言检测与触发

当发送方和接收方的`preferred_lang`不同时，系统自动触发人格保真翻译：

（1）检测发送方的`preferred_lang`和接收方的`preferred_lang`是否相同
（2）不同时，调用人格保真翻译引擎，以接收方的语言为目标语言
（3）翻译结果附带溯源元数据一同发送

支持14种语言：zh、en、ja、ko、fr、de、es、pt、ru、ar、hi、th、vi、id。

#### 3.2.4 独立人格保真翻译服务

提供独立的翻译API端点，用户可在发送消息前使用自己的人格画像对消息进行翻译：

```
POST /api/social/translate
{
  "content": "待翻译内容",
  "source_lang": "zh",
  "target_lang": "en"
}
```

确保对方收到的不是"翻译腔"而是"我本人如果会这门语言会怎么说"。

### 3.3 有益效果

（一）首次在跨语言社交中保留了说话人的个人风格，使翻译结果读起来像本人所写，而非机器翻译。

（二）在协议层面实现了翻译溯源，接收方可以同时看到原文和翻译，知道翻译是人格保真的。

（三）自动跨语言检测机制使用户无需手动操作即可享受人格保真翻译服务。

（四）独立翻译服务使用户可以主动控制自己消息的翻译风格。

## 四、摘要

本发明公开了一种基于数字分身人格画像的跨语言人格保真翻译及社交通信方法。该方法将用户的人格画像（性格特征、说话风格、惯用表达方式、幽默感、语气等）作为翻译提示词的组成部分注入AI翻译引擎，使翻译结果在完成语言转换的同时保留原始说话人的个人风格，而非生成通用机器翻译风格的内容。系统在接收到需跨语言投递的消息时，自动提取发送方用户的人格画像生成个性化翻译提示词，调用AI引擎执行风格感知翻译，并在消息中附加翻译元数据字段（目标语言、翻译置信度、人格一致性评分）。本发明解决了现有机器翻译在跨文化社交通信中丢失说话人个人风格、导致"翻译腔"的问题，将数字分身作为文化与语言的桥梁，使用户无论使用何种语言均能保持一致的个人表达风格，适用于多语言社交平台、跨文化商务通信、国际化AI代理服务等场景。

## 五、权利要求书

1. 一种基于用户人格画像的跨语言翻译方法，其特征在于：将用户的性格特征描述和说话风格描述作为翻译提示词的组成部分注入AI翻译引擎，使翻译结果保留原始说话人的个人风格特征（幽默感、语气、正式程度、惯用表达方式）。

2. 根据权利要求1所述的方法，其特征在于：翻译提示词框架包含用户名称、性格特征、说话风格、源语言和目标语言五个参数，指示AI引擎以用户本人的表达习惯进行改写而非逐词翻译。

3. 一种在社交消息协议中嵌入翻译溯源元数据的方法，其特征在于：每条翻译后的消息包含原始内容（original_content）、源语言（original_lang）、目标语言（target_lang）和翻译风格（translation_style）四个字段，使翻译过程完全透明可追溯。

4. 一种基于用户语言偏好差异的自动跨语言检测和触发机制，其特征在于：当消息发送方和接收方的preferred_lang字段值不同时，系统自动触发人格保真翻译流程，无需用户手动操作。

5. 一种独立的人格保真翻译服务方法，其特征在于：提供独立于消息发送流程的翻译API端点，用户可使用自己的人格画像对任意内容进行人格保真翻译。

## 五、在先技术对比

| 在先技术 | 与本发明的区别 |
|----------|---------------|
| Google Translate / DeepL | 通用翻译，无个人风格保留，无人格画像注入机制 |
| 微信翻译 / iMessage翻译 | 嵌入式通用翻译，无人格保真处理 |
| ChatGPT多语言模式 | AI通用多语言能力，不代表特定用户人格 |
| Meta SeamlessM4T | 语音翻译，无用户人格画像注入机制 |
| DeepL API | 提供翻译API但无基于用户性格画像的个性化翻译 |

本发明的独创性：首次将数字分身的人格画像用于跨语言翻译过程，使翻译结果保留说话人的个人风格，并在社交消息协议层面实现了翻译溯源的标准化嵌入。
