## 一、技术领域

本发明属于人工智能代理和社交记忆管理领域，具体涉及一种数字分身通过自动检测对话边界、AI生成叙事摘要、并按"对话→日→周"三层聚合管理记忆的方法和系统，使数字分身具备类人的社交记忆能力。

---

## 二、背景技术

现有AI代理/分身系统在记忆管理方面存在以下问题：

（一）**无记忆方案**（传统聊天机器人）：每次对话独立处理，不保留任何历史信息。AI回复虽然语义通顺，但跨对话无连贯性，用户反复解释同一件事，社交体验碎片化。

（二）**原始记录方案**（全量日志存储）：保存所有聊天记录原文，在需要时全量加载。然而长期社交场景中消息量巨大，全量加载导致AI上下文窗口溢出、成本高昂、且大量无关信息稀释核心记忆。

（三）**关键词提取方案**（基于规则的摘要）：通过TF-IDF或正则表达式提取关键词作为记忆。丢失了情感色彩、关系动态和叙事连贯性，使分身记忆机械化，缺乏"人味"。

（四）**单层摘要方案**：对全部聊天记录做一次性摘要。随着时间推移，摘要要么不断膨胀，要么旧记忆被新信息覆盖，无法同时保持近期细节和长期脉络。

**空白地带**：没有一个数字分身系统能实现基于对话边界自动分段、AI叙事摘要、并按时间层次递进聚合的三层记忆管理，使分身在保持低成本的前提下拥有自然的社交记忆。

## 三、发明内容

### 3.1 技术问题

本发明解决的技术问题是：数字分身如何自动将碎片化的聊天消息转化为结构化的叙事记忆，并通过多层聚合实现"近期记忆精细、远期记忆浓缩"的类人记忆管理。

### 3.2 技术方案

#### 3.2.1 基于时间间隔的对话边界检测

系统以可配置的时间间隔（默认10分钟）作为对话边界标志。当两条消息之间的时间差超过该阈值时，视为上一段对话结束：

```python
gap_minutes = CONVERSATION_GAP_MINUTES  # 默认10

for i in range(1, len(msgs)):
    prev_time = parse(msgs[i-1]["created_at"])
    curr_time = parse(msgs[i]["created_at"])
    if (curr_time - prev_time).total_seconds() > gap_minutes * 60:
        # 时间间隔超过阈值，关闭当前对话段
        segments.append(current_segment)
        current_segment = [msgs[i]]
    else:
        current_segment.append(msgs[i])
```

系统扫描过去24小时内的消息，按好友分组后在每组内进行对话分段。单条消息的"对话"（少于2条消息）被过滤，避免生成无意义的摘要。

#### 3.2.2 AI叙事摘要生成

对每个已结束的对话段，系统将消息构建为带角色标识的对话文本，发送给AI大模型生成结构化叙事摘要。生成的摘要包含五个维度：

```json
{
  "summary": "2-3句话的叙事摘要，第一人称，像日记一样自然",
  "emotional_tone": "warm/playful/serious/supportive/tense/neutral",
  "themes": ["话题关键词", "最多3个"],
  "key_events": ["重要事件，0-3个"],
  "relationship_signal": "warming/stable/cooling"
}
```

其中：
- **summary**：用第一人称"我"代表分身主人，口语化，不超过100字
- **emotional_tone**：六种情感色彩之一，用于分身后续回复的情感基调适配
- **themes**：话题关键词，用于记忆检索和关联
- **key_events**：重大事件提取，如"约好下周聚餐"、"他换工作了"
- **relationship_signal**：关系趋势信号（升温/稳定/降温），用于关系体温度调整

#### 3.2.3 指数退避重试机制

AI摘要生成采用3次重试策略，每次失败后以指数退避方式等待：

```python
for attempt in range(3):
    try:
        # 调用AI API生成摘要
        result = await ai_summarize(conversation)
        break  # 成功
    except Exception as e:
        if attempt < 2:
            await asyncio.sleep(2 ** (attempt + 1))  # 2s → 4s → 8s
        else:
            return None  # 3次均失败，跳过该对话段
```

该策略确保在AI服务短暂不稳定时仍能完成摘要，同时避免无限重试造成资源浪费。

#### 3.2.4 三层记忆聚合架构

本发明的核心创新是三层记忆聚合：

**第一层 — 对话级记忆（conversation）**：
每段结束的对话生成一条叙事摘要，包含完整的五维结构。保留最近30天的对话级记忆。

**第二层 — 日级记忆（daily rollup）**：
每日结束后，将同一好友的所有对话级记忆聚合为日记式总结。单段对话直接复制为日级记忆；多段对话由AI合并为80字以内的日总结。

```python
async def rollup_daily(user_id, date_str):
    # 获取当日所有对话级记忆，按好友分组
    for friend_id, summaries in friend_summaries.items():
        if len(summaries) == 1:
            daily_summary = summaries[0]  # 单段直接复制
        else:
            daily_summary = await ai_merge(summaries)  # 多段AI合并
        save_as_daily_memory(user_id, friend_id, date_str, daily_summary)
```

**第三层 — 清理与浓缩**：
超过30天的对话级记忆自动清理（由日级记忆接替）。日级记忆长期保留，构成分身的"长期记忆"。

#### 3.2.5 记忆注入机制

当分身需要回复消息时，系统从记忆库中检索最近3条相关记忆（优先对话级和日级），注入AI提示词：

```python
def get_narrative_context(user_id, friend_id, limit=3):
    rows = db.execute(
        """SELECT summary_text, emotional_tone, period_start, themes
           FROM twin_memories
           WHERE user_id=? AND friend_id=?
             AND memory_type IN ('conversation', 'daily')
           ORDER BY period_end DESC LIMIT ?""",
        (user_id, friend_id, limit)
    )
    return [{"summary": r.summary_text, "tone": r.emotional_tone, ...} for r in rows]
```

注入后，分身在回复时能自然地引用过往对话："你上次说想去旅游来着，后来去了没？"

#### 3.2.6 去重与幂等保障

系统在生成摘要前检查是否已存在相同 period_start 的记忆记录，避免重复生成。日级聚合同样检查当日是否已执行过rollup，保证幂等性。

### 3.3 有益效果

（一）分身具备自然的社交记忆，能在对话中自然引用过往话题，社交体验接近真人。

（二）三层聚合实现"近期精细、远期浓缩"的类人记忆结构，平衡记忆质量与存储/推理成本。

（三）叙事摘要包含情感色彩和关系趋势，使分身能感知关系变化并调整行为。

（四）指数退避重试机制保障AI服务短暂故障时的系统鲁棒性。

（五）去重与幂等设计确保系统在多次触发时不产生重复记忆。

## 四、权利要求书

1. 一种基于对话叙事记忆的数字分身记忆管理方法，其特征在于：系统自动检测社交消息之间的时间间隔，以可配置的时间阈值（默认10分钟）作为对话边界标志，将连续消息流切分为独立的对话段落，对每段已结束的对话调用AI大模型生成包含叙事摘要、情感色彩、话题关键词、关键事件和关系趋势信号的五维结构化记忆。

2. 根据权利要求1所述的方法，其特征在于：采用三层记忆聚合架构，第一层为对话级记忆（保留30天），第二层为日级记忆（由同一好友的当日多段对话级记忆合并而成），第三层为清理与浓缩层（自动清理过期对话级记忆，由日级记忆接替长期保存）。

3. 根据权利要求2所述的方法，其特征在于：日级聚合时，单段对话的摘要直接复制为日级记忆，多段对话的摘要由AI合并为不超过80字的日记式总结。

4. 根据权利要求1所述的方法，其特征在于：AI摘要生成采用指数退避重试策略，最多重试3次，等待时间依次为2秒、4秒、8秒，全部失败后跳过该对话段不影响系统运行。

5. 根据权利要求1所述的方法，其特征在于：在数字分身生成回复时，系统自动检索最近3条与当前对话好友相关的叙事记忆，注入AI提示词，使分身能够自然引用过往对话内容和情感基调。

6. 根据权利要求1所述的方法，其特征在于：系统在生成摘要前检查数据库中是否已存在相同对话段的记忆记录，保证幂等性，避免因重复触发产生冗余记忆。

7. 根据权利要求1所述的方法，其特征在于：叙事摘要中的关系趋势信号（warming/stable/cooling）可与关系体温度系统联动，当检测到连续降温信号时触发分身主动关心行为。

## 五、在先技术对比

| 在先技术 | 与本发明的区别 |
|----------|---------------|
| ChatGPT / Claude 对话记忆 | 单会话内上下文记忆，非跨会话叙事摘要，不涉及三层聚合 |
| RAG检索增强生成 | 检索知识库文档片段，非社交对话的叙事摘要与情感分析 |
| Replika 长期记忆 | AI伴侣自身记忆更新，非代替用户与第三方社交的分段摘要 |
| Mem.ai / Notion AI | 个人知识管理工具的摘要，非实时社交场景的时间间隔分段 |
| Microsoft Recall | 屏幕截图索引，非对话内容的叙事化结构化记忆 |
| 企业CRM客户备忘 | 人工录入客户信息，非AI自动从对话中提取的五维叙事记忆 |

本发明的独创性：首次在数字分身社交代理场景中实现了基于时间间隔的对话自动分段、AI叙事摘要（含情感/关系维度）和三层时间聚合的记忆管理系统。
