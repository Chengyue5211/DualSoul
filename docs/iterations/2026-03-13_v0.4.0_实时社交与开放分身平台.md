## 二、本次迭代内容

### 2.1 WebSocket 实时消息修复

**问题**: 好友发的消息和分身自动回复在对方界面看不到。

**根因**:
- WebSocket 消息匹配只检查 `from_user_id === chatFriend.user_id`，忽略了分身回复的 `to_user_id` 匹配
- `appendIncoming` 不区分"我的分身回复"和"对方的消息"

**修复**:
```javascript
// Before: if(d.from_user_id === chatFriend.user_id)
// After:  if(d.from_user_id === chatFriend.user_id || d.to_user_id === chatFriend.user_id)
```

**影响文件**: `web/index.html`

---

### 2.2 分身自动回复逻辑重构

**问题**:
- 分身只在对方"离线"时自动回复，在线但不看页面时不回
- 分身和真人同时回复导致对方困惑
- "分身建议"功能无用且干扰体验

**解决方案**:
1. **去掉在线状态限制** — `twin_auto_reply` 开关控制一切，不再检查 `is_online()`
2. **新增"分身代聊"切换** — 聊天页面右上角实时切换按钮
3. **彻底删除分身建议** — 前端UI/CSS/JS + 后端endpoint + WebSocket事件全部清理

**用户体验**:
- 开关 ON = 分身接管，自动回复所有消息
- 开关 OFF = 真人自己聊，分身不干预
- 实时同步服务器设置，Profile页和聊天页状态一致

**影响文件**: `web/index.html`, `dualsoul/routers/social.py`

---

### 2.3 视觉AI集成（分身看图说话）

**问题**: 用户在分身聊天中发图片，分身完全无法理解图片内容。

**解决方案**: 集成通义千问VL视觉模型（qwen-vl-plus）

**技术实现**:
- 新增 `AI_VISION_MODEL` 配置项
- `twin_self_chat` 接受 `image_url` 参数（base64 data URL）
- 图片消息使用多模态消息格式：`[{type: "image_url"}, {type: "text"}]`
- 超时从8s增加到20s（视觉请求更慢）

**影响文件**: `dualsoul/config.py`, `dualsoul/twin_engine/responder.py`, `dualsoul/models.py`, `dualsoul/routers/social.py`, `web/index.html`

---

### 2.4 性别字段

**问题**: 分身不知道主人的性别，回答"我是无性别设定"。

**解决方案**:
- `users` 表新增 `gender` 字段
- Profile 页新增性别选择器（男/女/未设置）
- 分身 prompt 注入性别信息
- `TwinProfile.build_personality_prompt()` 自动包含 Gender 行

**影响文件**: `dualsoul/database.py`, `dualsoul/models.py`, `dualsoul/routers/identity.py`, `dualsoul/twin_engine/personality.py`, `web/index.html`

### 2.6 开放分身导入平台（核心战略升级）

**战略思考**:

DualSoul 的定位不是"养分身"，而是"让分身社交"。养分身的事情交给各个平台（年轮、OpenClaw、未来的其他平台），社交的事情在 DualSoul 完成。

```
培养平台 (年轮/OpenClaw/...)     DualSoul (社交舞台)
+----------+                    +----------+
| 投喂/训练 | --TPF v1.0导入--> | 本地分身 |
| 观察/记忆 | --增量同步-----> | + 社交层 |
| 骨架合成  | <--互动反馈----  | + 好友圈 |
+----------+                    +----------+
```

**数据架构 — 三张新表**:

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `twin_profiles` | 五维人格骨架 | 判断力/认知/表达/关系/边界 + 价值观 + 行为模式 |
| `twin_memories` | 记忆摘要 | 日/周/月/季/年 + 情绪基调 + 主题 |
| `twin_entities` | 实体识别 | 人/地/物/事/概念 + 重要度 + 关系网络 |

**API 设计**:

| 端点 | 功能 |
|------|------|
| `POST /api/twin/import` | 完整导入（首次） |
| `POST /api/twin/sync` | 增量同步 |
| `GET /api/twin/status` | 查看导入状态 |

**分身引擎升级 — 双轨prompt**:

- **本地分身** (twin_source='local'): 简单2行（personality + speech_style）
- **导入分身** (twin_source='nianlun'/'openclaw'/...): 丰富多维prompt
  - 五维人格画像 + 价值观排序
  - 最近5条记忆摘要
  - 重要度Top10实体（人/事/物）
  - 行为模式 + 表达风格 + 边界规则

**开放性**: `source` 字段不限定值，任何平台名都可以。只要按 TPF v1.0 格式导出，就能导入 DualSoul。

**影响文件**: `dualsoul/database.py`, `dualsoul/models.py`, `dualsoul/main.py`, `dualsoul/routers/twin_import.py`(新), `dualsoul/twin_engine/personality.py`, `dualsoul/twin_engine/responder.py`

## 三、设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 分身建议功能 | 删除 | 真实测试证明用户觉得无用且干扰 |
| 自动回复触发条件 | 不看在线状态 | 用户在线不等于在看聊天，toggle更直观 |
| 分身数据存储 | 热列+冷JSON | 五维数据需要频繁查询，原始包存冷JSON做审计 |
| 导入架构 | 开放source | DualSoul做社交平台，不绑死单一培养源 |
| 记忆单独成表 | 是 | 记忆是增长集合(可能数百条)，需高效查询"最近N条" |
| 实体关系 | 扁平JSON | DualSoul不需遍历关系图，嵌入JSON更简单 |
