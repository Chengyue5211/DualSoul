## 命名规范

文件名格式: `YYYY-MM-DD_vX.Y.Z_主题简述.md`

## 迭代文档应包含

1. **迭代背景** — 为什么做这次迭代
2. **具体改动** — 每个功能的问题、方案、影响文件
3. **设计决策** — 做了什么选择、为什么
4. **测试发现** — 真实用户测试暴露的问题
5. **数据统计** — commit数、代码行数、测试通过数
6. **下一步** — 未完成和计划中的工作
